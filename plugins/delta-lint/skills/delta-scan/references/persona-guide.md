# Persona Mode Guide

delta-lint のスキャン結果を、読み手に合わせた言葉で出力するモード。

## 3つのペルソナ

| ペルソナ | 対象ユーザー | 出力の特徴 |
|---------|------------|-----------|
| `engineer` | 開発者（デフォルト） | ファイルパス・メソッド名・行番号 |
| `pm` | PM・事業責任者・CEO | ユーザー操作ベース・仕様の穴・ビジネス判断 |
| `qa` | QA・品質管理チーム | テストシナリオ・再現手順・環境条件 |

## 使い方

### コマンドで指定

```
/delta-scan --for pm
/delta-scan --for qa
```

### 自然言語で指定

以下のような表現でもペルソナが自動判定される：
- 「PM向けで」「非エンジニア向けで」「わかりやすく」 → pm
- 「QA向けで」「テストケースにして」「テストシナリオで」 → qa
- 「技術的に」「エンジニア向けで」「詳しく」 → engineer

### デフォルトを変更

```
/delta-scan set-persona pm
```

`.delta-lint/config.json` に保存される。以降そのリポジトリでは pm がデフォルト。

## 仕組み

1. 検出エンジン（detector.py）は共通 — 全ペルソナで同じ findings を出す
2. persona_translator.py が `claude -p`（サブスク内CLI、$0）で翻訳
3. CLI が使えない場合はテンプレートベースのフォールバック
